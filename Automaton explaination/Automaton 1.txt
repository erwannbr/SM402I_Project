When we launch the main we got the following result. We need to choose which automaton we want to read. In our case we will choose to launch the first one. 

""
Which automaton do you want to read? 1

===================================
        Automaton Menu
===================================
1. Display automaton
2. Check properties (std / det / complete)
3. Standardize
4. Determinize & complete
5. Minimize
6. Complement
7. Test a word
8. Load another automaton
0. Exit
===================================
Choose an option: 
""

We got a menu, and we can choose differents choices that we are gonna see here: 

1. Display automaton 

We will get the following table representing the automaton. 

""
      |   a  |
--------------
 IT 0  |      |
""

In our case, this transition table where State 0 is marked as the starting point (Initial Transition). The empty cell indicates that there is no move defined for the machine when it receives the input character 'a'.


2. Checks properties

In this part we will get the differents properties of the automaton. (like standard, deterministic, complete)

""
Choose an option: 2
  Standard    : yes
  Deterministic: yes
  Complete    : yes
""

So in our case, the automaton is standard, deterministic, and complete


3. Standardize

In this part it will standardize the automaton

""
Choose an option: 3
Automaton is already standard — skipping.
"" 

Since it's already standard, it will simply skip the step. 


4. Determinize & complete

In this part, it will determinize and complete the automaton if it's not deterministic

""
Choose an option: 4
Automaton is already a complete DFA — nothing to do.
""

Since it's already deterministic it will skip this step also. 


5. Minimize

In this part, it's gonna, first minimize the automaton, and then display in a table the new automaton. 

""
  Partition at step 0:
    Group 0: ['0']
  -------------------
  State   | →a      |
  -------------------
  0       | --      |
  -------------------

  Partition at step 1:
    Group 0: ['0']
  -------------------
  State   | →a      |
  -------------------
  0       | --      |
  -------------------

The automaton is already minimal.

--- MINIMIZATION: state correspondence ---
  New state 0 ← ['0']

      |   a  |
--------------
 IT 0  |      |
""

Since it's already minimize it will skip this step also.


6. Complement

In this part it will display the complement of the choosen automaton. 

""
""

Nothing to display since there is only one state enter, and terminal. 

7. Test a word

In this part, we will try to give a word to the automaton to verify that this is the good one or to try words. 

""
Enter a word (or 'end' to stop): a
  'a' is rejected.
Enter a word (or 'end' to stop): 
  '' is accepted.
""

In our case, there is only one word accepted which is ' ', and all the other word like 'a' are rejected. 


8. Load another automaton

""
Which automaton do you want to read? 
""

We can choose to load another automaton.
